So here basically every element has to be adjacent to atleast one element greater than it. Thus for each element we will have to delete all the elements less than it on one side till it is adjacent to an element greater than it or has no element on that side anymore. So it is logical to start doing this from the max element of the entire array. But here we need to consider both the cases - should we delete all the lesser elements to the left and then continue to make the right subarray cool or vice versa? For this we need to use dp. Lets say dp[l,r]=min operations needed to make [l,r] cool. We are assuming the max element in this range is at k. Then our dp state becomes dp[l,r]=min(k-l+dp[k+1,r],r-k+dp[k-1,l]). But the problem is n here is too large to perform a 2d dp. So what we can do is to use the state dp[curr] where curr is the max element in the current subarray being processed. To handle this we need to use a Max Cartesian tree. Thus now this turns into a linear dp where the no of operations currently being done is equal to the size of the subtree and the dp value of the other range becomes the dp value of the other subtree. Basically here the subtree represents a range in the array thus optimising our dp to linear time.

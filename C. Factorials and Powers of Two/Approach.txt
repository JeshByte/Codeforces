Our first observation should be that there aren't that many powers of 2 or factorials in the given range. Also every number can be represented as a sum of powers of 2 - their binary representation. So in worst case that is our answer. We now need to think if there is any way to reduce the answer. Binary representation is done greedily, so there is no way to reduce the answer using more powers of 2. Thus we will need to use the factorials. Lets say we use some number f of factorials which create a sum of s, then the final answer for that case will be f + no_of_set_bits(n-s). Ignoring 1 and 2(as they would be already considered in the powers of 2 category), there are only 14 more valid factorials. So we can use bitmasking to go through all the 2^14 combinations and find the best cost out of all of those.

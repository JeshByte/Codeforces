Even though the answer is technically monotonic, we can't directly apply binary search on it as only a few specific numbers can be our answer. But what we can do is to run binary search on a constraint - the number within which we will try to keep all the subarray sums. So if we are able to keep all the subarray sums within this limit by splitting it into <=k subarrays, then the current max subarray sum for this constraint can be a possible answer and we will check whether it is better than our current answer. Obviously as the constraint is reduced, our answer will also get better. Also the reason <=k subarrays is allowed, is even if we split the current set of subarrays into smaller ones the new maximum will always be <= our current one. But on the other hand, for >k subarrays we will need to join the current set of subarrays which might increase the maximum subarray sum and due to which it would be invalid. At the end, the minimum answer found out of all the ones for each valid constraint is our final output.

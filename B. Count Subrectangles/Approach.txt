The way the input is given, the final rectangle will have smaller perfect rectangles of 1. Now what would be the dimensions of these smaller rectangles? If we dry run a few testcases, we will see that the length of continous segments of 1 in a is the number of rows in these rectangles. Similarly that of b is the number of columns in these rectangles. Multiplying them would give us the areas of all of these rectangles. But we won't do that as it would be very large. Now lets say we need to build a n*m rectangle within these smaller rectangles. So first we can only use those rows which are >= n and cols which are >=m. Lets say we find a r and c which satisfies this condition. So the number of n*m rectangles we can fit within the r*c rectangle will be (r-n+1)*(c-m+1). Now what if we want to find this value for all possible r and c with respect to a given n*m. Then the formula would be (rsum-n*rcnt+rcnt)*(csum-m*ccnt+ccnt), where rsum and csum are the sum of all the valid r anc c, and rcnt and ccnt is their total count. If we maintain a sorted array of all the possible r and c, we can easily calculate this value for a given n*m in little time using binary search and suffix sums. But how to find all the possible n*m? n and m will be valid only if they are factors of k. So we just need to find all of k's factors(n) and then find it's corresponding m. There would be at most 2*sqrt(k) such pairs. Then using the above given formula, we will find the number of possible rectangles for each of these n*m pairs and their total sum will be our final answer.

As the subarray size we need to check for is fixed, we can use a sliding window approach here. But we need to first see how to check whether two numbers will be in increasing order or not after applying the required operation. So we can see that if the 2nd number is already >= the 1st number it will be always greater after applying the operation as the 2nd number will always be multiplied by a 2 with higher power. Now lets consider the case where the 2nd number is lesser. We can see that whenever we apply the operation, irrespective of the powers of the 2 being multiplied, the 2nd number will try to close the gap by a number equal to its own value. So for two numbers n1 and n2, n2 can become greater than n1 on applying the operation only if n2>=(n1-n2+1). The +1 is because the number has to be strictly increasing. This will work for any power of 2 as the difference scales accordingly. Thus we also know that if a subarray fulfils this condition, even if we reduce the power of 2 for each term(which we will need to do to check for the next subarray), they would still fulfil the condition. Thus everytime for a new subarray we just need to check the new element with the last element of the previous subarray. So we can just apply a fixed sliding window algorithm to check for all the subarrays of size k+1 using the above proven conditions in linear time.

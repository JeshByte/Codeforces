Every tea at index i gets tasted by tasters starting from i to n. There would be an index j>=i, where till j-1 the tasters get a full tasting(their max capacity) and the jth taster gets a partial tasting i.e. what is remaining of the ith tea. The tasters after that get no tasting. How to get this j? We can do this by using upper bound on a prefix sum array of b. Now the next challenge is to keep track of the number of full tastings each taster gets. Like in the above case tasters from i to j-1 will all get a full tasting, thus that value will increase by +1 for all tasters in that range. Such range queries are answered in constant time using delta encoding. Using these techniques we will be able to get the answer in nlogn time which is within the given constraints.

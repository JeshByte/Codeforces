So it would never be optimal to have more than one component with same color. As then only the max valued one will be considered. So we would always like to color the tree such as for each color there is only one component. The number of colors range from 1 to n-1. Lets consider the case of n-1 colors. Here the optimal coloring will be to give each edge a unique color. Then the answer will be the sum of wt[i]*deg[i] for all nodes i. Now lets consider the case of n-2 colors. Here we have one less color. So 2 edges will need to share one color. As we already discussed we will like to do this in a manner such that each color has only one connected component. If we dry run a few test cases, we will observe that whenever we try to do so we basically reduce the value by wt[i] where i is a node with degree >= 2. So obviously to optimise it at every step, we will then go for the node with minimum weight with degree>=2. We can simulate this efficiently for all the number of colors using a min heap. Then we will get our answer in nlogn time.

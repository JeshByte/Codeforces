First, we need to realise that the order of deletion doesn't matter. Deleting a node, won't make it so that nodes which were deletable before aren't so anymore or vice-versa. So if we are able to get all the deletable nodes, sorting them would give us the right order. Now lets focus on getting these nodes. A node is deletable if the respect value is 1 for both it and it's immediate children. So we can simply apply dfs to find the deletable nodes using this condition in linear time.

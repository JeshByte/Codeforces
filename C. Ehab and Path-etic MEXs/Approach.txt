Lets say we want to ensure that the largest MEX in the tree is n. So we must label the edges such that there is no path which has all the values 0,1,....n because then the MEX would end up being greater than n. For this we need three seperate branches, where we can split the numbers 0,1,....,n to ensure that this doesn't happen. But if we are able to do this for a n, we can also do this for all values less than n till 2. Now when will we have three or more branches/splits in a tree? That will happen when a node has a degree of >= 3. Then you can arbitrarily label three of its edges as 0,1,2. In this way there would be no path where all these values would be together and MEX will remain at 2. And if there is no such node with a degree >=3, you can label all the edges randomly. This is because in such a situation, there would be atleast one path consisting of all the edges where the MEX will always be n-1. So we just need to check for these 2 conditions to get the optimal way to label the edges.

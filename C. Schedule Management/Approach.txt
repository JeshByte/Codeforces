Our first observation should be that the final time taken is monotonic. If all tasks can be completed at t, then they can also be completed at t+1 and so on. So we can try to apply binary search on answer here. So we need to check whether all the tasks can be completed within time t. As the workers work independently, we can look at this another way - does each worker complete its tasks within time t? Now how to assign tasks to a worker. We will obviously first assign it the tasks it is proficient in. So the number of tasks it is proficient in which it can do are complete=min(t,cnt) where cnt is the no of tasks it is proficient in. Then the no of tasks which it isn't proficient in which it can do in the remaining time are others=floor((t-complete)/2). And also the no of tasks that it is proficient in which it couldn't complete are incomplete=cnt-complete. If we maintain a running sum of others and incomplete for each worker, then at the end if others >= incomplete -> t is valid or else it isn't. Thus in this way we will find the answer in n*logm time. Also the minimum answer is 1(this will happen for example when n=2 and the tasks are 1 2) and the maximum answer is 2*m(acts as a reasonable upper bound assuming each task is done by a worker who isnt proficient in it but the final answer will always end up being less than it).

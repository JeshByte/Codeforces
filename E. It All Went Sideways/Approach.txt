First we will reverse the array for ease of computation as the elements will always shift to the right. Now to calculate the answer for a given array we will need to maintain a running minimum. At every step update the mininum and the contribution from that column will be arr[i]-minimum. Now what would change if the height of a column is reduced by 1? If we dry run a few test cases, we will see that the contributions of the columns before the affected one remains the same. Also the contributions change only upto the next smaller column from the affected one. So their contributions also remain the same. What changes for the columns in between? The running minimum could change for these columns. So we will just accordingly update it with respect to the affected column's new height and calculate the new contributions for these columns. We will need to precompute prefix sums of the column heights, suffix sums of the initial contributions and the next smaller elements for the columns to calculate these updated values in constant time. The max value out of all the cases will be the final answer.

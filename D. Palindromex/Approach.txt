Lets say we have a palindromic sequence of mex m. There will be smaller palindromic sequences within it too. But there is no point in checking them as their mex will be <=m and we only care about maximising our mex. Also each element occurs only twice. So when we are at an element, we will already know where this hypothetical palindromic sequence will end if it exists. We will then check the elements inside. If all of them are ok then the entire sequence is palindromic and we will check it's mex and update our answer accordingly. Then we will start checking from the element after the current sequence's end. If not, then we will continue checking from the first element which breaks our current palindromic sequence. We will then get our optimal answer using this sliding window-like approach.

So, if we simplify this question, for each k, we need to find the sum of the biggest prefix of length l from the students in each university arranged in descending order such that l%k==0. But doing this directly is inefficient. So first we will process our input to make further calculations easy i.e. divide the students into their universities, arrange them in descending order and then store them in the form of a prefix sum array. So now if we take a prefix of length l from any of the universities. Then for each factor f of l, the prefix sum of elements from (l-f)th index and lth index will be part of the answer for k=f. Thus we will just iterate through all the students university wise, then for each index we will find the factors and add their prefix contribution to the required k's. This can be done in n*sqrt(n) time which is well within the given time bounds.

The value of r with respect to k is a decreasing/monotonic funtion, so we can apply binary search to find it. But we will then need an efficient way to find the AND value of a subarray. If we observe closely, for a subarray only those bits which are set in all the elements in the subarray contribute to the final sum. So we can use a prefix sum array which will maintain the prefix sum of jth bit till ith element. Then for subarray [l,r], the jth bit will contribute to the AND value only if pre[r][j]-pre[l-1][j]=r-l+1. Using this we can quickly calculate the AND value of a subarray by just iterating through the bits and use it for our binary search logic. This way we will get our optimal answer for all the queries. The time complexity will be - q*log(n)*32.

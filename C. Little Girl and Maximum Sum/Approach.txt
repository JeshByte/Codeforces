So basically if we go through all the ranges and find the frequency of occurence of all the indices, we could just greedily assign the highest value to the index with highest frequency and so on. But how to find the frequency of the indices efficiently? We can sort the given segments in descending order of their end points. Then for each index(n -> 1), we will go till the last segment with end point >= the index. During this traversal we will also insert their start points into a pbds ordered set. Then when we reach the required segment, we will find the number of start points in the set less than the index which will give us the number of segments within which the index occurs. Using this 2 pointer approach we would be able to find the frequency for all the indices in linear time. The rest is pretty straight forward, after which we will have our final answer. We will also have to make it so that the set is able to handle duplicate elements.

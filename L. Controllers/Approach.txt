Let n1 and n2 be the two numbers. Let a and b be the number of times n1 and n2 is pressed for plus. c and d are same for minus. Then the final number can be expressed as n1*(a-c)+n2*(b-d). Here there can be 3 cases for this expression to be 0 - 
- Both the terms are 0. So a-c=0 and b-d=0 which means the number of plus and minus are equal. So if they are equal, the answer is always YES.
- Both the terms have same absolute value but the first one is positive and the second one is negative. Thus n1*(a-c)=n2*(b-d). This will happen only for multiples of lcm of n1 and n2. Lets say this value is lcm*const. Thus a-c=lcm*const/n1 which further reduces to a-c=k1*const where k1=lcm/n1. Thus a=c+k1*const. Similarly d=b+k2*const. Number of plus = c+k1*const+b and minus = c+b+k2*const. This reduces to plus-minus=const*(k1-k2). Thus for this to be valid k1-k2 must divide plus-minus and plus needs to be >=k1*const and minus needs to be >=k2*const.
- Both the terms have same absolute value but the first one is negative and the second one is positive. This is basically like the previous case but opposite. The condition for const is same, just here plus needs to be >=k2*const and minus needs to be >=k1*const.
If the query satisfies any of these conditions, the answer is YES. Otherwise the answer is NO.

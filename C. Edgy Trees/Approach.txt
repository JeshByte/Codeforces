Lets try to find the number of bad paths. Then we can simply subtract them from the total number of possible paths to get the answer. For a path to be bad, it must only consist of nodes between which there are no black edges. So basically they will be a connected component with only red edges. Thus what we can do is to use DSU and just consider the red coloured edges to find such components. Then for each component, lets say it's size is s. Then the number of bad paths that can be made out of it are s^k. We will subtract these values for each component from the total to get the final answer. We would also need to use binary exponentiation to calculate these values fast.

So first of all MEX here is monotonic. Thus we can apply binary search on answer here. Now how to verify that we can get a mex of say m? For that, we need 0,1,...m-2,m-1 to occur in the array at least once. So first we will see which of these elements are already present in the array. Then we will know what elements are required and what extra elements are left in the array to be transformed using the given operation. So by modding an element with any number we can get values between 0 to Ceil(m/2)-1 and m itself. Thus we can see that the larger the number, the more values it can generate. So a greedy strategy can be used here. So we will sort the required and the extra values. And then see if the required values can be generated using the extra values or not. If we get all the required values, then a mex of m is possible. Otherwise it isn't. This checking logic works in n*logn time. Thus our final solution will work in n*(logn)^2 time, which is within the given constraints.

First we need to observe that when it comes to making a decision of placing a block on a tower, we just need to compare it with the topmost block. So it simplifies our problem a bit as we will need to keep track of only the top blocks of all the towers made till now. Now whenever we will try to place a block, we should try to place it on the tower whose topmost block is the least number greater than the current block out of all the towers. This is because we can't place it on blocks less than or equal to it and placing it on a block too big is wasteful as we could have placed bigger blocks on them in the future. If there is no such block available then we will need to create a new tower. We can use upper bound function to find the optimal block to place it on efficiently which we will run on our vector of current towers. We can use a vector here and not a set as the vector will always automatically remain sorted due to the way the blocks are being placed(which can by verified by dry running a few test cases). And at the end, the size of this vector will be our answer which we found in n*logn time.

So lets treat the couples as points with x and y coordinates. A point can be paired with another one if both of it's x and y coordinates aren't equal to the current point's x and y coordinates. So which are the ones that can't be paired? Points which have the same x-coordinate or the same y-coordinate. And what overlap will they have? The current point itself. So for point (x,y), the number of possible points to pair with becomes k-(xc[x]+yc[y]-1) where xc[i] is number of points with x-coordinate equal to i and yc[i] works similarly. And every time we have calculated the answer for a point we will remove it to ensure the other point's contribution doesn't include it anymore. We can do it by reducing xc[x] and yc[y] and k by 1 after every iteration. So in this way, summing the unique individual contribution of each point will give us the answer. We can use frequency arrays for xc and yc.

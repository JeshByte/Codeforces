With every operation we are reducing the total sum of the array by 2. So to make it a zero array, the total sum definitely needs to be even. But this alone isn't enough. On top of this, the maximum element(maxi) needs to be <= the sum of the remaining elements(rsum). Let's go through this case by case - 
- maxi=rsum. Here you could take all the other elements and apply the operation on them with maxi, till they all become 0. So answer would be YES.
- maxi>rsum. Here even if you take all the other elements to reduce the maxi, it still won't reach zero. So answer would be NO.
- rsum>maxi. The total sum is rsum+maxi, which should be even. So both of them have to be even or odd. Thus there difference would be even. So I would be able to make them equal by performing operations within the remaining elements after which as shown above the answer would be YES. But how can I guarantee I would be able to always apply enough operations to make them equal? To apply the operation, there needs to be atleast 2 positive numbers among the remaining elements. There can never be a single positive element because as rsum>maxi, it consisting of one element will mean maxi isn't the maximum element which is a contradiction. Thus we would always be able to satisfy the condition here, so the answer would be YES.

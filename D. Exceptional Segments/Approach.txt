So lets say we have an array prefix which has the prefix xor of the given series. Then the l,r can be expressed as prefix[l-1]^prefix[r]. This needs to be equal to 0. Thus prefix[l-1]=prefix[r]. l-1 will range from 0 to x-1 and r will range from x to n, thus ensuring that x always remains within the segment. Now if we calculate the prefix xor of the series till a certain number, we will observe a pattern. The answers are cyclic, with the pattern n,1,n+1,0 and so on. n and n+1 will never be equal on either side of x, so we just need to find l-1s whose xor is 0 and same for the rs. We will do the same thing for l-1s and rs with xor value 1. Using that we will get the final answer. This count can be found easily using some basic arithmetic. There is just one edge case which is that l-1 goes till 0 whose prefix xor is 0 too. So we need to seperately handle this one while finding the l-1s with 0 xor value. Using these values and by applying the given mod, we will get our solution.

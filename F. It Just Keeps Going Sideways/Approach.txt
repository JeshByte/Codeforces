We will reverse the given array as the blocks always shift to the right and it will make further processing simpler. Now we can calculate the answer for a given array in this manner -> Go level by level from 0 to max column height. Let's define sum(x) as the sum of values of numbers -> 0,1,...x-2,x-1. At every level i, the distance of the blocks moved will be (sum(n)-sum of indices of columns with heights i,i-1,i-2,....)-(sum(cnt)). Here, cnt=n-sum of frequency of columns of heights i,i-1,i-2,... The sum of all these values will give you the total answer. Now what will change if a column's height is reduced from x to x-1? If we dry run a few test cases we will observe that nothing changes in the contribution of level x. But for level x-1, we will need to add the index of the affected column(which had height of x before the operation) and remove it's sum(cnt) and instead add sum(cnt-1). Through this way, we will get the answer for each of the cases. The max one will be our final output. We will need to use some preprocessing to calculate the updated values fast.

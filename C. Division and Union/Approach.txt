Basically we need to find all the intersecting segments. If all of them are intersecting then the answer would be -1. We will first sort the segments in descending order of their endpoints. Now we know that segments after it, end before it. So for them to intersect with you their end point needs to be >= your starting point. Lets say you traverse such a segment. If it's starting point is greater than your starting point then its okay. Otherwise, you should be using this new starting point for comparison with the later segments. Thus we will maintain a running minimum of the starting points and use the above logic to find all the intersecting segments and mark them accordingly. As I already mentioned, if all of them are intersecting/marked the answer is -1. Otherwise put the marked segments into one set and the unmarked ones into the other set. This will give us a valid answer.

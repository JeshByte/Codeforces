So lets first dry run the problem, and see how from a digit we get new digits until we end up with an already visited one. Then we will notice that the even digits starting from 2 form a cycle(2->4->8->6->2) and the odd digits after applying the operation once lead to this same cycle. The only exceptions are 0 and 5. 0 can't generate new digits and with 5, after the first operation you are stuck with the 0. So to make it uniform, apply the operation once on numbers with odd digits. Then if the array has any number with 0 as last digit, then all the numbers must be equal to it. Otherwise it is impossible to convert the array into the desired one. After handling this case, now our array will have only elements with even digits starting from 2. Lets take 22. Also we will be calling numbers even/odd depending on the parity of its 10th place digit. So with 22, we will see that it can generate even numbers ending with 2/4/8 and odd numbers ending with 6. This can be seen with all even numbers ending with 2/4/8. This same logic can be extended for the other possible type of numbers. Using this we can infer whether the final array can be converted or not.

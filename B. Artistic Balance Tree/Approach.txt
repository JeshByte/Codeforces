Using the given operation, an element can be swapped only with elements in positions of same parity. So when you have to perform the operation x, you can mark any of the elements in positions with same parity as x. So lets say we have x odd operations and y odd operations. Then we will mark as many positive integers with these operations and then could repeat them too to not end up marking any negative numbers which would further optimise our answer. But we would still need to mark at least one element for each of the types of the operation. So we would need to handle the case of all negative numbers seperately.

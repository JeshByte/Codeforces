Basically we need to divide the tree into as many even sized components as possible. This won't be possible for trees with odd number of nodes. So for them, we will output -1. Now lets focus on the even sized trees. As these are trees, initially they would form one single component. So we can do a dfs dp where for a node, we find the sizes of all of it's subtrees. If the subtree is of even size we cut it off and increment our answer, otherwise we keep it. Then we would pass the sum of the sizes of the retained subtrees + 1(to account for the current node itself), to its parent. In this way we will be able to find the answer in linear time. The base condition will be to return 1 for the leaf nodes. This recursive algorithm will give us our final answer within the given constraints.

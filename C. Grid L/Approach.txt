For a n*m rectangle, the number of unit segments needed will be n*(m+1)+m*(n+1). This will need to be equal to p+2*q. Now you could fit the p unit segments whichever way you want to. But there are some restrictions on how we could fit the q L-shaped segments. To handle that we could try to find the maximum q segments(qmax) that could fit in it. Then we will know that our q is valid if it is <=qmax as the remaining could be covered using the p segments. So for a n*m shape(assuming n<=m), we can see by dry running a few testcases that qmax=n*(n+1)+n*(m-n). So if we have n and m, we can verify using this whether they could be made using the given p and q. But how to find n and m? p and q can go till 1e8, so p+2*q will go till 3e8. Assuming n=m, from the formula n*(m+1)+m*(n+1)=p+2*q we can see that n can't go beyond 1e5. So we can just go through all possible values of n. For each n, try to find a valid m if any. And then try to check whether it could be constructed using the given p and q. All this could be easily done using the above derived forumlae.

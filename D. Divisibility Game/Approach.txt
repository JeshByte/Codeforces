So lets say there are certain numbers Alice can delete and certain numbers that Bob can delete and then there are the ones that both can delete. So optimally, both of them would try to delete the common ones. Then they would delete the numbers from their individual set for the later moves. This can be easily simulated. Lets now focus on getting the numbers that can be deleted by Alice and Bob respectively. Alice can delete numbers who are divisible by atleast one element of a. So we can go for a sieve style approach, generating all the multiples of each element in a and if it is present in b, then it can be deleted by Alice. Now on the other hand, Bob can delete elements that are not divisible by at least one element of a. So which numbers can Bob can't delete? -> Numbers that are divisible by every element of a. For that to happen, it needs to be divisible by lcm of all elements of a. Thus we will first calculate this lcm, then elements of b which are not divisible by the lcm can be deleted by Bob. All of this will work within the given time limit as it is given that elements of a and b are <=(n+m).

If we dry run a few test cases, we will observe that the higher x you accumulate and use, the more operations you save. So what we can do is to put all the elements in a set along with a unique id to differentiate between duplicate elements. Then in every iteration extract the max and min element. Now there are 2 cases. The first case is max and min are different elements. In such a case, we will take as much of min as possible till our x is <= max. If our x reaches max, then using it we will delete the max element too. After this we will accordingly update and reinsert/delete the min and max element. The other case is where our min and max are the same element val(basically our set has only 1 element remaining). Here we will use the formula floor((val-x)/2) to get how much n we can take from the element while ensuring that x remains <= (val-n). After this calculation we will use the same logic as above, to update our values and the set. We will also keep track of x and the total number of operations through out the iterations. We will keep on doing this till the set becomes empty. Then the number of operations will be our final answer.

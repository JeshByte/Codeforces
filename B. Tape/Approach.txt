So if there was no constraint on k, the optimal answer would have been to use 1 length strips on all the segments resulting in a total answer of n. But as this is always not going to be the case, we need to choose which segments we will reduce to 1 like this. So as we are allowed only k strips, its obvious that we should try to reduce the k-1 biggest segments to 1 and then use the final strip to cover the remaining segments. Using this greedy approach, will give us our answer in nlogn time.
